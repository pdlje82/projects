Required

Using GridSearch without Cross_validation would lead to overfitting, because the classifier would not be tested or optimized on unseen data.

But it's possible to optimize the hyperparameters by comparing their results against a single validation set. What would the advantage of cross-validation over this method?

As mentioned in the previous review:

  • When we pick the best instance of the model at the end of this validation process with a single validation set, what do we have?
    • A model that generalizes well to unseen data?
    • Or a model that fits the validation set well?

Comment

I want to make sure that there is no confusion between train, validation, and test sets:

  • The train test is used to find the model's parameters.
  • The validation test is used to calibrate the model's hyperparameters - the parameters that the model does not learn from the data (like the maximum depth of a decision tree in this project).
  • The tests set is used to evaluate the final model's performance - the score on the test set is the one we are ultimately interested in.

In particular, cross-validation s a way of using the same data set for both training and validating our model. But it's still best practice to keep a truly held-out data set for testing purposes after the model is trained and validated.
